# SOLUTION

## Где лежит state

Я держал три вида состояния раздельно:

- **Текущая заявка — в URL.** Беру id из `location.pathname` через `matchPath` в
  `WorkspaceShell`. Отдельного `currentTicketId` в Redux нет: иначе было бы два источника
  и рассинхрон.
- **Последние заявки — в `localStorage`** (`recentTicketStorage.ts` поверх `safeStorage`).
  Это пользовательский контекст.
- **Данные и статусы заявок — в RTK Query** (`ticketsApi`). Это единственный источник
  о том, какие заявки есть и в каком они статусе.

Черновик оператора и тема — обычный локальный state компонента/контекста.

## Граница palette / ticket

`src/app/command-palette` — универсальный движок: знает только про `CommandDefinition[]`,
ничего про тикеты не импортирует. Ticket-команды собираются снаружи, в
`workspaceCommands.ts`: туда из `WorkspaceShell` приходят `currentTicket` и колбэки, а
наружу отдаётся готовый массив для палитры.

Зависимость односторонняя: feature знает про палитру, палитра про feature — нет.
Команды, завязанные на заявку, попадают в список только когда `currentTicket !== null`,
поэтому видны лишь на странице карточки.

## Edge cases

`readRecentTicketIds` не ломается если недоступный `localStorage`, пустое значение,
битый JSON, не-массив и массив с мусором (фильтрую до непустых строк). Запись обёрнута в
`try/catch`. Если id из storage уже нет в ответе API, заявка просто не показывается — фильтрую на месте, в `TicketListPage`.
Команда «Mark reviewed» становится `disabled`, если заявка уже проверена.

## Trade-offs

- Recent-блок обновляется за счёт того, что `TicketListPage` перемонтируется при возврате
  со страницы заявки. Живой подписки на `localStorage` не делал — для этого сценария
  избыточно.
- Фильтрацию устаревших id делаю в списке, а не в storage: storage не должен знать про API.
- Для уже проверенной заявки команду показываю `disabled`, а не прячу — так оператору
  понятнее.
- В integration-тесте пришлось обернуть дерево в `AppThemeProvider` (его нет в
  `renderWithProviders`, а `WorkspaceShell` зовёт `useAppTheme`). Менять общий харнесс ради
  одного теста не стал.
